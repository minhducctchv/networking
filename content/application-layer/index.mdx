# Chương 6: Application Layer

Chương này bước lên đỉnh của ngăn xếp mạng — nơi các **application** (ứng dụng — phần mềm mà người dùng và lập trình viên tương tác trực tiếp) gặp gỡ mạng máy tính. Sau khi hiểu Transport Layer (Chương 5) đưa dữ liệu end-to-end giữa hai tiến trình, ở đây bạn sẽ học các protocol lập trình viên dùng hằng ngày: DNS để phân giải tên, HTTP/HTTPS (từ 1.1 đến 3), các giao thức realtime như WebSocket/SSE/gRPC, HTTP caching + CDN, và SMTP/FTP/SSH cho email, truyền file và truy cập từ xa.

## Mục tiêu học tập

- Mô tả quá trình **DNS resolution**, phân biệt recursive/iterative query, và nêu mục đích các record type (A, AAAA, CNAME, MX).
- Phân tích cấu trúc **HTTP** request/response, phân loại đủ 5 nhóm **status code**, và giải thích vai trò **TLS** trong **HTTPS**.
- Hiểu **HTTP/2** (multiplexing, HPACK) và **HTTP/3** (QUIC, connection migration); biết khi nào dùng phiên bản nào.
- Chọn đúng công cụ realtime: **WebSocket** (hai chiều), **SSE** (một chiều, auto-reconnect), **gRPC** (RPC hiệu năng cao).
- Áp dụng **HTTP caching** đúng (`Cache-Control`, `ETag`, `Vary`) và hiểu **CDN** đưa cache tới gần user thế nào.
- Giải thích mục đích, port, và use case cho **SMTP**, **FTP** và **SSH**.

## Kiến thức tiên quyết

Bài học giả định bạn đã nắm các chương trước. Từ **Chương 1** bạn cần khái niệm protocol và mô hình phân lớp. Từ **Chương 4** bạn cần IPv4/IPv6 vì DNS là cầu nối giữa tên miền và địa chỉ. Quan trọng nhất, từ **Chương 5** bạn cần port, socket, và sự khác biệt giữa TCP và UDP — mỗi application protocol trong chương chạy trên TCP hoặc UDP tại một port xác định.
