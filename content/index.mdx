# Networking cho Software Developer

Đây là tài liệu học **computer networking** (mạng máy tính — hệ thống các thiết bị kết nối với nhau để trao đổi dữ liệu) được viết bằng tiếng Việt, dành riêng cho **software developer** muốn nắm vững nền tảng mạng để xây dựng, vận hành và gỡ lỗi hệ thống một cách tự tin. Nội dung đi từ những khái niệm nền tảng nhất cho tới các chủ đề hiện đại như HTTP/3, service mesh, tail latency, và distributed tracing.

## Định hướng: học mạng theo góc nhìn dev

Khác với sách giáo khoa mạng truyền thống, khóa này **cắt bỏ có chủ đích** các chi tiết ít giá trị cho developer (chi tiết mã hóa đường truyền vật lý, tính CRC bằng tay) và **thêm mạnh** những mảng dev đối mặt hằng ngày:

- **HTTP/2, HTTP/3, WebSocket, gRPC, SSE** — protocol thực tế bạn dùng khi viết API và client.
- **HTTP caching + CDN** — pattern hash-in-filename, ETag, Cache-Control cho asset và API.
- **CORS, cookies (SameSite/Secure/HttpOnly), CSRF** — web security mà browser thực thi.
- **Certificate trong thực tế** — Let's Encrypt, ACME, cách debug lỗi cert phổ biến.
- **Service mesh, mTLS, Ingress, NetworkPolicy** — kiến trúc mạng Kubernetes hiện đại.
- **Networking từ code (Node/Go/Python)** với timeout, retry, connection pool.
- **curl cookbook** — công thức debug API hằng ngày.
- **Latency & tail latency, timeouts + retries + circuit breakers, observability** — kỹ năng vận hành hệ ở scale.

## Mục tiêu sau khóa học

Bạn sẽ có thể:

- Hiểu cách dữ liệu di chuyển qua **OSI/TCP-IP**, tính **subnet**, cấu hình routing cơ bản.
- Giải thích cơ chế **TCP**, **UDP**, **DNS**, và các phiên bản **HTTP/1.1/2/3**.
- Chọn đúng công cụ realtime: **WebSocket** vs **SSE** vs **gRPC**.
- Áp dụng **HTTP caching** đúng cách, hiểu **CDN** hoạt động thế nào.
- Áp dụng **web security** (CORS, cookies an toàn, CSRF token, CSP) cho API và browser.
- Nhận diện các mối đe dọa mạng, xin và quay vòng cert **Let's Encrypt**.
- Thiết kế **NetworkPolicy** và **Ingress** cho Kubernetes; hiểu **service mesh + mTLS**.
- Viết HTTP client với **timeout, retry, circuit breaker** đúng cách.
- Đo và tối ưu **tail latency**, thiết kế **observability** với metrics + traces + logs.
- Sử dụng thành thạo **Wireshark, tcpdump, ping, traceroute, dig, curl, ss** để chẩn đoán.

## Đối tượng người đọc

Khóa học phù hợp với **software developer**, **DevOps/SRE**, và sinh viên IT đã quen với việc lập trình và dòng lệnh, chưa có nền tảng vững về mạng. Bạn không cần kiến thức mạng trước đó — mọi khái niệm đều được giải thích ngay khi xuất hiện.

## Quy ước ngôn ngữ

Toàn bộ phần giải thích viết bằng **tiếng Việt**, nhưng các thuật ngữ kỹ thuật giữ nguyên bằng **tiếng Anh** — ví dụ `subnet mask`, `three-way handshake`, `Cache-Control`. Lần đầu một thuật ngữ xuất hiện, nó kèm chú thích tiếng Việt ngắn. Mọi đoạn code, câu lệnh và file cấu hình đều giữ nguyên tiếng Anh.

## Cách đọc hiệu quả

- **Từ đầu tới cuối** cho người mới: mỗi chương xây trên chương trước.
- **Đi thẳng vào một chủ đề** nếu có nền tảng: các bài đều liệt kê rõ kiến thức tiên quyết.
- **Đọc code + chạy công cụ**: mỗi bài có ví dụ `curl`, `dig`, `tcpdump`, code Node.js/Go/Python — chạy trực tiếp là cách học nhanh nhất.
- **Làm bài tập**: mỗi bài có bài tập lý thuyết và tình huống với đáp án. Cố tự giải trước khi mở đáp án.
